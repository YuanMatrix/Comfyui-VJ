## 🤔 为什么看起来"没有日志"？

### 原因 1: 请求日志只在有请求时显示

当服务器启动后，如果没有 API 请求，就不会有新的日志。

**只有当您发送请求时，才会看到：**
```
============================================================
📥 收到请求: POST /api/generate-video
   来源: 127.0.0.1
   Content-Type: application/json
============================================================
```

### 原因 2: 终端可能被关闭或切换

如果您切换到其他窗口或关闭了终端，就看不到日志了。

---

## 🔧 解决方案

### 方案 1: 使用新窗口启动（推荐）

**双击运行:**
```
启动VJ_API_新窗口.bat
```

这会在独立窗口中运行 VJ API，您可以随时查看日志。

### 方案 2: 在当前终端运行

```bash
cd vj
.\start_vj_api.bat
```

**保持终端打开**，所有日志都会实时显示。

### 方案 3: 查看实时日志文件

VJ API 的日志会显示在终端中。如果您想保存日志，可以：

```bash
cd vj
.\start_vj_api.bat > vj_api_log.txt 2>&1
```

然后查看 `vj_api_log.txt` 文件。

## 🧪 测试日志是否工作

### 发送测试请求，查看日志

```bash
# 健康检查（会显示请求日志）
curl http://localhost:5002/health

# 查看所有任务（会显示请求日志）
curl http://localhost:5002/api/tasks

# 查看特定任务（会显示详细日志）
curl http://localhost:5002/api/tasks/{task_id}
```

每次请求都会在终端中显示：
```
============================================================
📥 收到请求: GET /health
   来源: 127.0.0.1
   Content-Type: None
============================================================
127.0.0.1 - - [08/Feb/2026 18:22:54] "GET /health HTTP/1.1" 200 -
```
